# 2026-06-02 工作日志

## 修复腾讯云 OCR 功能

### 问题
- `vite.config.ts` 手写 TC3-HMAC-SHA256 签名存在拼写错误（Credential/SignedHeaders/canonicalRequest），Write/Edit 工具持续无法正确写入，导致 Vite 500 错误
- 旧 Vite 进程（5178端口）未清理，前端请求发到错误端口
- `ImageToExcelPage.tsx` 第 437 行 `value || ;` 语法错误
- 腾讯云官方 SDK 返回数据没有 `Response` 包装层，但前端代码在找 `data.Response`
- base64 数据可能含换行符导致 `XLSX.read` 解析失败

### 修复步骤
1. 安装 `tencentcloud-sdk-nodejs` 官方 SDK，彻底绕开手写签名
2. 重写 `vite.config.ts` 中间件，改用 SDK 的 `ocr.v20181119.Client`
3. 中间件把 SDK 返回包成 `{ Response: data }` 格式，兼容前端代码
4. 修复 `ImageToExcelPage.tsx` 语法错误 `value || ;` → `value || ''`
5. 添加 base64 数据清理 `dataB64.replace(/[\s\r\n]/g, '')`
6. 杀掉占用 5178 端口的旧 Vite 进程

### 验证结果
- 腾讯云 OCR API 调用成功，返回 `Response.Data`（base64 xlsx）
- 前端正确解析并显示 34行 × 10列 的表格
- 识别精度：印刷体表格有一定错误（|→1, o→0, 逗号多余），手写表格效果更好

### 文件修改
- `vite.config.ts` — 中间件改用腾讯云官方 SDK
- `src/pages/ImageToExcelPage.tsx` — 语法错误修复 + base64 清理
